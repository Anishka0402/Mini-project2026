MEMBER 1 — ANISHKA JAIN

IoT System Architecture & AI/ML Risk Assessment Engine | Contribution: 33.3%

Member 01

Anishka Jain

IoT System Architect & AI/ML Engineer
Project Contribution: 33.3%  |  Module: IoT Sensing Layer + AI Risk Engine
Domain Focus

IoT Hardware Integration, AI/ML Model Development, Sensor Data Engineering, Edge Computing, Time-Series Analysis

Python TensorFlow/Keras MQTT/CoAP Raspberry Pi Scikit-learn XGBoost Edge AI LoRaWAN
CORE RESPONSIBILITIES
IoT Architecture DesignDesign complete sensor network topology for heritage site monitoring. Select sensors: DHT22 (temperature/humidity), MQ-series (air quality), accelerometers (vibration), UV sensors, crack displacement sensors. Define MQTT/CoAP protocols and edge preprocessing pipelines for remote sites.
Sensor Hardware SetupConfigure Raspberry Pi / Arduino IoT nodes. Program sensor firmware in MicroPython/C++. Implement edge-level noise filtering, data normalization, and local buffering. Set up LoRaWAN/Wi-Fi wireless communication for heritage sites with limited connectivity.
Data Preprocessing PipelineBuild data ingestion pipelines from IoT nodes to cloud. Handle missing sensor values, outliers, and time-series alignment. Create feature engineering scripts extracting deterioration-relevant features: temperature cycling stress, humidity gradients, vibration frequency signatures.
AI Deterioration ModelDevelop and train supervised ML models — Random Forest, LSTM (time-series), XGBoost — to predict stone/wood/metal deterioration rates from environmental exposure history. Use historical conservation records + synthetic simulation data. Target: AUC greater than 0.88 on validation set.
Vandalism Risk DetectionBuild anomaly detection module using Isolation Forest to flag unusual activity patterns: unauthorized access, sudden structural shocks, night-time vibration anomalies. Integrate detection output with Bhoomika's alert pipeline via REST API calls on risk threshold breach.
Model EvaluationConduct cross-validation, confusion matrix analysis, and feature importance ranking. Generate Risk Prediction Performance Report with AUC, precision, recall, F1-score. Document hyperparameter tuning process. This is a required project expected outcome.
Cloud IntegrationCollaborate with Bhoomika to ensure IoT streams are correctly ingested via AWS IoT Core. Test end-to-end flow: sensor node to cloud to AI prediction output. Expose trained model as REST API endpoint consumed by both backend (for alerts) and dashboard (for visualization).

Key Deliverables

  • IoT Sensor Network Architecture Diagram
  • Configured sensor nodes with firmware code
  • Data preprocessing & feature engineering scripts
  • Trained AI deterioration prediction model
  • Vandalism anomaly detection module
  • Risk Prediction Performance Report
  • Model training notebooks & evaluation metrics
  • REST API endpoint for AI predictions

Technologies & Tools

  • Python 3.x — primary language
  • TensorFlow / Keras — LSTM deep learning
  • Scikit-learn — Random Forest, Isolation Forest
  • XGBoost — gradient boosting
  • MQTT / CoAP — IoT communication
  • Raspberry Pi 4 / Arduino — edge hardware
  • LoRaWAN — long-range wireless
  • AWS IoT Core — cloud IoT gateway

MEMBER 2 — BHOOMIKA AGARWAL

Cloud Infrastructure, Backend REST API & Real-Time Alert System | Contribution: 33.3%

Member 02

Bhoomika Agarwal

Cloud Architect & Backend API Developer
Project Contribution: 33.3%  |  Module: Cloud + Backend API + Alert System
Domain Focus

Cloud Infrastructure Design, RESTful API Development, Database Engineering, Real-time Event Systems, Cybersecurity, CI/CD Automation

AWS/Azure FastAPI/Node.js PostgreSQL Docker/K8s Apache Kafka CI/CD JWT/RBAC Redis
CORE RESPONSIBILITIES
Cloud Architecture DesignDesign and deploy secure, scalable cloud architecture on AWS. Set up VPCs, IAM roles, auto-scaling groups, and load balancers. Services include: AWS IoT Core (MQTT ingestion), RDS PostgreSQL (structured data), TimescaleDB (time-series sensor data), S3 (raw sensor dumps), Lambda (serverless event triggers).
Database Schema & ManagementDesign relational database schema for sensor readings, site metadata, AI predictions, alert logs, and user accounts. Implement TimescaleDB for efficient time-series storage of high-frequency sensor data. Set up automated backups, read replicas, data retention policies, and partitioning for performance.
RESTful API DevelopmentBuild comprehensive FastAPI / Node.js REST API with endpoints for: sensor data ingestion, AI model result retrieval, alert management, user authentication, site management, and automated report generation. Full OpenAPI/Swagger documentation. Versioned endpoints for future extensibility.
Real-time Alert SystemBuild event-driven alert pipeline using Apache Kafka / AWS SNS+SQS. When Anishka's AI model flags a risk, trigger multi-channel notifications: email (AWS SES), SMS (Twilio), push notifications, and in-app alerts. Implement four severity levels: Low, Medium, High, Critical with escalation logic.
Security & AuthenticationImplement JWT-based authentication and Role-Based Access Control (RBAC) for conservation authorities, government users, and field operators. Set up HTTPS/TLS encryption, API rate limiting, SQL injection prevention, input validation, and audit logging. Ensure full OWASP Top 10 compliance.
Data Pipeline OrchestrationSet up Apache Airflow / AWS Step Functions to orchestrate daily batch jobs: model retraining triggers, automated report generation, data archival tasks, and database maintenance. Monitor pipeline health with CloudWatch. Alert on pipeline failures within 5 minutes.
Integration & Load TestingIntegrate Anishka's AI prediction API results into the backend pipeline. Provide stable, documented API endpoints to Shraddha for dashboard consumption. Write integration and load tests using Locust — ensuring system handles 500+ concurrent IoT device connections without performance degradation.

Key Deliverables

  • Cloud architecture diagram (AWS/Azure)
  • Database schema — ER diagram + migrations
  • REST API with full Swagger documentation
  • Real-time multi-channel alert system
  • JWT authentication + RBAC system
  • CI/CD pipeline (GitHub Actions)
  • Load testing report (500+ connections)
  • Security audit & compliance report

Technologies & Tools

  • FastAPI (Python) / Node.js — backend
  • PostgreSQL + TimescaleDB — database
  • Apache Kafka — event streaming
  • Docker + Kubernetes (EKS)
